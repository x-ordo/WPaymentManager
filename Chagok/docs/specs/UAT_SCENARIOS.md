## 📌 0. 개요 (Overview)

본 문서는 CHAGOK 시스템이 **실제 변호사 업무 환경**에서 의도한 비즈니스 가치(Time Saving, Calm Control)를 제공하는지 검증하기 위한 최종 관문 테스트 시나리오입니다.

- **테스트 주체:** 내부 QA팀 및 베타 테스터 변호사 3인
- **테스트 환경:** Staging (Prod와 동일한 스펙)
- **성공 기준(Exit Criteria):** 모든 P0 시나리오 Pass 및 변호사 만족도 점수 4.0/5.0 이상

## 🧪 1. P0: 핵심 가치 검증 (Core Value Loop)

*서비스의 존재 이유인 '시간 단축'과 '대량 처리 안정성'을 검증합니다.*

### UAT-01: 대량 증거 일괄 처리
- **상황:** 변호사가 사건 초기 증거 수집 단계에서 50개 이상의 파일을 한 번에 업로드함.
- **사용자 행동:**
  1. 사건 상세 화면 진입.
  2. 탐색기에서 50개의 혼합 파일(사진 30장, 녹취 5개, PDF 15개)을 드래그앤드롭.
- **검증 포인트 (Pass/Fail):**
  - [ ] UI가 '멈춤(Freezing)' 없이 즉시 '업로드 중' 상태로 전환되는가?
  - [ ] 개별 파일의 진행률(Progress bar)이 시각적으로 인지 가능한가?
  - [ ] 5분 이내에 모든 파일의 상태가 '분석 완료'로 변경되는가?
  - [ ] 실패한 파일이 있을 경우, 명확한 재시도 버튼이 노출되는가?

### UAT-02: 타임라인 정합성
- **상황:** 사건의 핵심 쟁점일인 '2023년 12월 25일'의 증거를 찾고자 함.
- **사용자 행동:**
  1. 타임라인 탭 클릭.
  2. 날짜 필터에 `2023-12-25` 입력.
- **검증 포인트:**
  - [ ] 해당 날짜의 메타데이터(EXIF 등)를 가진 증거만 필터링되는가?
  - [ ] 오전/오후 시간 순서대로 정확히 정렬되어 있는가?
  - [ ] 카카오톡 캡처 이미지의 경우, 이미지 내 대화 날짜를 인식하여 배치되었는가?

### UAT-03: 소장 초안 생성 및 활용
- **상황:** 정리된 증거를 바탕으로 초안을 작성하여 법원에 제출할 문서를 만듦.
- **사용자 행동:**
  1. 'Draft Preview' 패널에서 '초안 생성' 클릭.
  2. 생성 완료 후 `.docx` 다운로드 버튼 클릭.
  3. 로컬 PC에서 MS Word로 파일 열기.
- **검증 포인트:**
  - [ ] '청구취지', '청구원인' 등 법률 서면 양식이 깨지지 않고 유지되는가?
  - [ ] 본문 내 `[증거 1-1]`과 같은 인용 부호가 실제 증거 리스트와 일치하는가?
  - [ ] AI가 작성한 문장이 비문 없이 자연스러운가?

## 🛡 2. P1: 리스크 방어 (Safety & Guardrails)

*법적 사고 및 데이터 오남용을 방지하기 위한 안전장치를 검증합니다.*

### UAT-04: 무(無)증거 초안 생성 시도
- **행동:** 증거가 하나도 없는 빈 사건에서 '초안 생성' 버튼 클릭.
- **기대 결과:**
  - [ ] API 호출이 차단됨.
  - [ ] "분석할 증거가 없습니다. 증거를 먼저 업로드해주세요."라는 안내 메시지 출력.

### UAT-05: 민감 데이터 삭제 (잊혀질 권리)
- **행동:** 실수로 업로드한 개인적인 사진을 '삭제' 버튼으로 제거.
- **기대 결과:**
  - [ ] "정말로 삭제하시겠습니까? 이 동작은 되돌릴 수 없습니다." 확인 모달 출력.
  - [ ] 삭제 후 UI 리스트에서 즉시 사라짐.
  - [ ] (Admin) S3 원본 경로 및 OpenSearch 인덱스에서 데이터가 실제 삭제되었는지 로그 확인.

### UAT-06: AI 환각(Hallucination) 체크
- **행동:** 채팅이나 검색창에 증거에 없는 내용(예: "남편이 해외 도박을 했어?")을 질문.
- **기대 결과:**
  - [ ] "제공된 증거 내에서 해당 사실을 확인할 수 없습니다."라고 답변.
  - [ ] 없는 사실을 지어내거나 추정하여 답변하지 않음.
