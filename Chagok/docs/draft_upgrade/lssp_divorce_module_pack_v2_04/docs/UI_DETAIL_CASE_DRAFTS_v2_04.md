# detail/case — Drafts Tab Spec v2.04

## 탭 구성
- [증거 정리] : 파일/폴더 + 상태(미검토/검토완료/증거적합) + 키포인트 추출
- [핵심포인트] : Keypoint 리스트(READY/DRAFT/CONTESTED) + Ground 매핑
- [초안] : 문서 타입 선택 → 자동 생성 → 블록 편집 → 내보내기

## ‘초안’ 탭 UX
1) 문서 타입 선택 카드
- 이혼소장 / 조정신청서 / 준비서면 / 증거목록 / 재산목록 / 양육계획서 / 협의서(협의이혼)

2) 생성 버튼
- “근거 기반 초안 생성(1차)” 버튼
- 생성 전 표시:
  - Ground 선택(G1~G6)
  - 사건일/인지일 입력(제척기간 계산에 사용)
  - 사건 타임라인(자동 생성 이벤트 표시)

3) 결과 화면(블록 에디터)
- Section 단위로 접기/펼치기
- 각 블록 우측:
  - Coverage score
  - Citation chip(증거/키포인트 링크)
  - “근거 보기” 클릭 → EvidenceExtract로 이동

4) 경고/알림 영역(상단 고정)
- 마감 기한 임박/경과(협의이혼 3개월 유효기간, 재판상 신고 1개월 등)
- 제척기간 임박/경과(G1/G6 등) 경고
- 누락된 필수 서류/합의서 경고

5) 내보내기
- v1: DOCX export
- v2: HWP export (hwpers/hwp-bridge)

## 품질 게이트(제출 방지)
- Coverage < 70 또는
- TODO 블록 3개 이상 또는
- Citation 없는 블록 존재
→ “제출용 문서로 부적합” 배너 + 누락 항목 바로가기.
