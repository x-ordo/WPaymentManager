# v2.12 Legal Search / Recommend UI (detail/case)

구성
- 상단: 추천 카드(법령/판례) + "초안에 삽입" 버튼
- 좌측: 검색 입력 + 필터(근거 G1~G6, 절차 단계, 태그)
- 중앙: 결과 리스트(요지/태그/점수/추천 이유)
- 우측: 상세 패널(스니펫, 적용요소, '이 블록에 삽입' CTA)

UX 규칙
- 추천은 '이유'가 반드시 1개 이상 보여야 한다(Explainable).
- 초안 삽입 시, 블록이 어떤 Claim에 해당하는지 먼저 선택/확인.
- Citation은 최소 단위(조문항/판례번호/스니펫)로 핀포인트 저장.
