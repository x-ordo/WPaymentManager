## 🎯 핵심 문제의식

> **"AI로 단순히 초안 작성하는 것은 차별화가 안 된다"**

현재 CHAGOK의 AI 기능은 초안 생성뿐이라 특별함이 없음.
**시각화**와 **실시간 피드백**으로 차별화된 가치를 제공해야 함.

## 💎 아이디어 1: 재산분할 실시간 시각화 (핵심 아이디어)

### 배경

- 이혼 시 **가장 중요한 것은 재산분할**
- 이혼 전문 변호 서비스를 쓰는 이유: **내 돈을 지키거나 상대방에게 더 받기 위해**
- 현재 문제: 증거가 재산분할에 얼마나 영향을 주는지 **즉각적으로 알 수 없음**

### 핵심 컨셉

```
┌─────────────────────────────────────────────────────────┐
│                  재산분할 예측 Dashboard                  │
├─────────────────────────────────────────────────────────┤
│                                                         │
│     [배우자 A]                    [배우자 B (의뢰인)]     │
│        45%  ◀━━━━━━━━━━┃━━━━━━━━━━━▶  55%              │
│                        ┃                               │
│     ████████████████████░░░░░░░░░░░░░░░░░░░████████    │
│                                                         │
│     총 재산 추정액: 5억 2천만원                          │
│     예상 분할액: 약 2억 8,600만원                        │
│                                                         │
├─────────────────────────────────────────────────────────┤
│  [+] 새 증거 업로드 시                                   │
│                                                         │
│  📄 "상대방 외도 증거 (카카오톡)" 업로드됨               │
│                                                         │
│     45% → 42%          ┃           55% → 58%           │
│     ▼ 3%p 감소          ┃           ▲ 3%p 증가          │
│                                                         │
│  💡 "외도 증거로 인해 유책배우자 판정 가능성 증가"        │
│     "유사 판례 기준 위자료 추가 청구 가능"               │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 사용자 시나리오

```
1. 변호사가 사건 생성, 기본 재산 정보 입력
   - 부동산, 예금, 주식, 퇴직금 등

2. 재산분할 Dashboard 표시
   - 초기 상태: 50% vs 50% (기본값)
   - 총 재산 추정액 표시

3. 증거 업로드할 때마다:
   - AI가 증거 분석
   - 해당 증거가 재산분할에 미치는 영향 계산
   - **실시간으로 % 바 애니메이션**
   - 변화 이유 설명 (판례 기반)

4. 변호사가 시각적으로 확인:
   - "이 증거를 넣으니까 3%p 유리해졌네"
   - "어떤 증거가 더 필요한지" 직관적 이해
```

### 영향 요소 (재산분할 비율 변화 기준)

| 요소 | 영향 방향 | 예상 변화폭 |
|------|----------|-------------|
| 외도/부정행위 증거 | 유책배우자 불리 | ±3~10%p |
| 폭언/폭행 증거 | 유책배우자 불리 | ±2~5%p |
| 경제적 기여도 증거 | 기여자 유리 | ±5~15%p |
| 양육 기여도 증거 | 기여자 유리 | ±3~10%p |
| 재산 은닉 증거 | 은닉자 불리 | ±5~10%p |
| 혼전 재산 증거 | 해당자 유리 (분할 제외) | 별도 계산 |

## 🗺️ 아이디어 2: 타임라인 마인드맵

### 참고 이미지
![타임라인 마인드맵](../KakaoTalk_20251203_175558774.jpg)

### 배경
- Google NotebookLM 스타일
- 증거/이벤트를 **날짜별 트리 구조**로 시각화
- 한눈에 사건의 흐름 파악 가능

### 핵심 컨셉

```
                        [사건 시작]
                            │
        ┌───────────────────┼───────────────────┐
        │                   │                   │
   [2024-01]           [2024-03]           [2024-06]
        │                   │                   │
   ┌────┴────┐         ┌────┴────┐         ┌────┴────┐
   │         │         │         │         │         │
[폭언1]  [폭언2]    [외도발견]  [통장]    [별거]   [이혼요구]
   │                   │
 카톡증거          사진증거
```

### 사용자 시나리오

```
1. 변호사가 증거들을 업로드

2. AI가 각 증거의 날짜/시간 추출

3. 타임라인 마인드맵 자동 생성:
   - 날짜별 분기
   - 유책사유별 색상 구분
   - 클릭 시 해당 증거로 이동

4. 변호사가 한눈에 사건 흐름 파악:
   - "3월에 외도 발견 후 6월에 별거했네"
   - "1월~3월 사이 증거가 비어있네"
```

### 예상 작업 범위

| 담당 | 작업 내용 | 상세 | 난이도 |
|------|----------|------|--------|
| **AI Worker (L)** | 날짜/시간 추출 | 증거에서 타임스탬프 파싱 | 🟢 (기존 기능) |
| **AI Worker (L)** | 이벤트 분류 | 증거 → 이벤트 타입 매핑 | 🟡 중간 |
| **Backend (H)** | 타임라인 API | `GET /cases/{id}/timeline-tree` | 🟢 낮음 |
| **Frontend (P)** | 마인드맵 UI | D3.js 또는 React Flow 기반 | 🔴 높음 |
| **Frontend (P)** | 인터랙션 | 줌, 팬, 클릭 이벤트 | 🟡 중간 |

### 우선순위
🟡 **중간** - 시각화 기능 확장

### 논의 필요 사항

- [ ] 마인드맵 라이브러리 선정 (D3.js? React Flow? GoJS?)
- [ ] 노드 클릭 시 동작 (증거 상세? 팝업?)

## 🔗 아이디어 3: 인물관계 그래프

### 참고 이미지
![인물관계 그래프](../KakaoTalk_20251203_175558774_01.jpg)

### 배경
- 옵시디언 지식 그래프 스타일
- 사건 관계자들을 **노드-엣지 그래프**로 시각화
- 복잡한 인물 관계 한눈에 파악

### 핵심 컨셉

```
                    [친구]                    [직장동료]
                   (노란색)                    (파란색)
                      │                          │
                      │                          │
    [시부모] ─────── [배우자] ═══════════════ [의뢰인] ─────── [친정부모]
   (빨간색)         (빨간색)      결혼        (초록색)         (초록색)
                      │                          │
                      │ 외도                     │
                      ▼                          │
                   [제3자]                     [자녀]
                  (분홍색)                    (보라색)

    ─── 관계선 (가족/친구/직장 등)
    ═══ 핵심 관계선 (부부)
```

### 사용자 시나리오

```
1. 변호사가 증거 업로드 + 사건 정보 입력

2. AI가 증거에서 인물 추출:
   - 카카오톡: 대화 참여자
   - 사진: 등장 인물 (태그)
   - 진술서: 언급된 인물

3. 인물관계 그래프 자동 생성:
   - 색상: 관계 유형별 (가족/친구/직장/제3자)
   - 선 굵기: 관계 강도/빈도
   - 클릭 시 해당 인물 관련 증거 필터

4. 변호사가 관계 파악:
   - "배우자가 이 사람과 자주 연락했네"
   - "시댁 식구들이 어떻게 연결되어 있는지"
```

### 노드 색상 분류 (예시)

| 색상 | 분류 | 예시 |
|------|------|------|
| 🔴 빨간색 | 상대방 측 | 배우자, 시부모, 시댁 친척 |
| 🟢 초록색 | 의뢰인 측 | 의뢰인, 친정부모, 친정 친척 |
| 🟡 노란색 | 친구 | 양측 친구 |
| 🔵 파란색 | 직장 | 직장 동료, 상사 |
| 🟣 보라색 | 자녀 | 자녀들 |
| 🩷 분홍색 | 제3자 | 외도 상대, 기타 관계자 |

### 예상 작업 범위

| 담당 | 작업 내용 | 상세 | 난이도 |
|------|----------|------|--------|
| **AI Worker (L)** | 인물 추출 (NER) | 증거에서 인물명 추출 | 🟡 중간 |
| **AI Worker (L)** | 관계 추론 | 인물 간 관계 타입 분류 | 🔴 높음 |
| **Backend (H)** | 인물 데이터 모델 | 인물/관계 스키마 | 🟡 중간 |
| **Backend (H)** | 그래프 API | `GET /cases/{id}/relationship-graph` | 🟡 중간 |
| **Frontend (P)** | 그래프 UI | Force-directed graph 시각화 | 🔴 높음 |
| **Frontend (P)** | 필터/인터랙션 | 노드 클릭, 관계 필터 | 🟡 중간 |

### 우선순위
🟡 **중간** - 시각화 기능 확장
